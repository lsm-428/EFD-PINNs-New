# 数据采样策略演进

## 📊 数据生成策略改进总结

### **核心改进概述**

从 `pinn_20260205_174333` 到 `pinn_20260418_130834` 的关键改进在于**时间采样策略的根本性优化**，这是导致训练损失从 34.59 降至 34.28 的主要原因。

### **改进详情**

#### **1. 时间采样策略升级**

**旧版本 (20260205)**:
- 使用均匀时间采样或分段采样
- 时间域内数据点分布均匀
- **问题**: 无法有效捕捉电润湿响应的早期快速动态变化

**新版本 (20260418)**:
```python
def sample_continuous_times(n_samples):
    """
    连续时间采样 (Beta 分布)

    使用 Beta(0.5, 1.0) 分布，自然地在早期 (t=0) 附近采样更多点，
    同时平滑覆盖整个时间域，避免人为的分段点 (如 15ms) 造成的密度突变。
    """
    # Beta(0.5, 1.0) 类似于 1/sqrt(x)，在 0 处概率密度高，向右递减
    # 这非常适合模拟暂态过程（变化集中在早期）
    t_samples = np.random.beta(0.5, 1.0, n_samples) * self.t_max
    return t_samples
```

**改进效果**:
- ✅ **Beta(0.5, 1.0)分布**: 在 t=0 附近采样密度更高
- ✅ **物理合理性**: 电润湿响应在早期变化最快，需要更密集采样
- ✅ **平滑过渡**: 消除了人为分段点造成的数据密度不连续

#### **2. 数据场景分布优化**

**旧版本**: 固定的场景比例和离散电压点采样

**新版本**: 优化的连续采样策略
```python
# 1.1 稳态数据 (40%) - V_from = V_to
n_steady = int(n_interface * 0.4)
V_steady = np.random.uniform(0, 30.0, n_steady)  # 连续电压采样

# 1.2 升压响应 (30%) - 0 -> V
n_up = int(n_interface * 0.3)
V_up = np.random.uniform(1.0, 30.0, n_up)

# 1.3 降压响应 (30%) - V -> 0
n_down = n_interface - n_steady - n_up
V_down = np.random.uniform(1.0, 30.0, n_down)
```

**关键改进**:
- ✅ **连续电压采样**: 从离散电压点改为连续电压范围 [0, 30V]
- ✅ **优化场景分布**: 40%稳态 + 30%升压 + 30%降压
- ✅ **统一时间采样**: 所有场景使用相同的 Beta 分布时间采样

#### **3. 边界条件数据改进**

**旧版本**: 简单的边界条件采样

**新版本**: 与界面数据一致的混合场景采样
```python
# 混合场景采样：40% 稳态, 30% 升压, 30% 降压
n_bc_steady = int(n_bc * 0.4)
n_bc_up = int(n_bc * 0.3)
n_bc_down = n_bc - n_bc_steady - n_bc_up
```

**改进效果**:
- ✅ **场景一致性**: 边界条件与界面数据采用相同的场景分布
- ✅ **时间采样统一**: 边界数据也使用 Beta 分布时间采样
- ✅ **物理完整性**: 更全面的边界条件覆盖

#### **Loss 改善量化**

#### **改进机制**

1. **更好的早期动态捕捉**
   - Beta分布使 t=0 附近数据点更密集
   - 更好地学习电润湿的快速初始响应
   - 提高了模型对暂态过程的学习能力

2. **更平滑的时间过渡**
   - 消除了分段采样造成的人为边界
   - 模型学习到更连续的时间演化过程
   - 改善了时间导数项的学习精度

3. **更丰富的场景覆盖**
   - 连续电压采样覆盖 [0, 30V] 全范围
   - 40/30/30 的场景分布更贴近实际应用
   - 提高了模型的泛化能力
