# EFD3D 重构报告 — 2026-05-28

## 概述

本次重构解决了模型"跑偏"问题：没学到不同电压下油膜状态，没学到 uvw 速度场。根本原因是物理约束冗余、计算量大、以及 EW 力公式存在量纲 bug。

## 2. 计算优化 — 统一梯度计算

### 2.1 新增统一梯度计算方法

在 `PhysicsConstraints` 中新增 `_compute_all_gradients()` 方法，**一次性**计算所有一阶和二阶梯度：

- **一阶梯度**：`u_x, u_y, u_z, u_t, v_x, ..., phi_x, phi_y, phi_z, phi_t`（共 20 个分量）
- **二阶梯度**：`lap_u, lap_v, lap_w, lap_phi, phi_xx, phi_yy, phi_zz, phi_xy, phi_xz, phi_yz`（共 10 个）
- 打包成 `grads` dict 传给所有子方法

### 2.2 重构所有活跃子方法

以下方法改为接收 `grads: Dict[str, torch.Tensor]` 参数，不再独立调用 `torch.autograd.grad`：

| 方法 | 改动 |
|------|------|
| `compute_navier_stokes_residual` | 从 grads 读取 20 个一阶+二阶梯度 |
| `_compute_vof_residual` | 从 grads 读取 phi 的一阶+二阶梯度 |
| `compute_electrowetting_residual` | 从 grads 读取 phi 一阶梯度 |
| `compute_interface_energy_residual` | 从 grads 读取 phi 一阶梯度（计算 \|∇φ\|） |
| `compute_laplace_pressure_residual` | 从 grads 读取 phi 一阶+二阶梯度 |
| `compute_sidewall_contact_angle_residual` | 从 grads 读取 phi 一阶梯度 |
| `compute_wall_wetting_residual` | 不改（不计算梯度） |
| `_compute_temporal_smoothness` | 不改（需要 model 做三时间点前向） |
| `_compute_dielectric_charge_residual` | 不改（不计算 phi 梯度） |
| `_compute_contact_line_dynamics_residual` | 从 grads 读取 phi 一阶+时间导数 |
| `_compute_top_boundary_residual` | 从 grads 读取 u_z, v_z |
| `_compute_unified_wetting_bc` | 不改（备选路径，默认关闭） |
| `compute_volume_conservation_residual` | 不改（统计量，无需梯度） |

### 2.3 回退兼容

所有子方法保留 `grads=None` 回退路径：当 `grads` 为 None 时（兼容直接调用），独立计算梯度。

### 2.4 预期效果

- 每个 epoch 的 autograd 调用从 **5+ 次减少到 1 次**
- 训练速度预期提升 **3-4 倍**

## 3. 物理公式修正

### 3.1 NS 中 EW 力公式（✅ 用户确认）

**旧代码 bug**（`constraints.py` L292-297）：
```python
interface_indicator = 4 * phi * (1 - phi)  # ❌ 无量纲，max=1.0
f_ew_x = f_ew_magnitude * interface_indicator * z_decay * dir_x
```

**问题**：
- `4φ(1-φ)` 无量纲，f_ew 量纲变为 `[N/m]` 而非 `[N/m³]`
- EW 力幅值小 ~10⁶ 倍，电润湿驱动力几乎无效

**修正**：
```python
# f_ew = ½ C_ew V_eff² * z_decay * ∇φ
# 量纲: [N/m] * [1/m] = [N/m³] ✅
# 方向: 沿 ∇φ（从水指向油），极性液体推动油墨向外
f_ew_x = f_ew_magnitude * z_decay * phi_x
f_ew_y = f_ew_magnitude * z_decay * phi_y
f_ew_z = torch.zeros_like(f_ew_x)
```

**物理验证**（用户确认）：
- 方向正确：沿 ∇φ（从水指向油），极性液体推动油墨向外
- 量纲正确：`phi_x = ∂φ/∂x` 有 `[1/m]` 量纲，`f_ew_magnitude * phi_x` = `[N/m] * [1/m]` = `[N/m³]`

### 3.2 z_decay 衰减长度（✅ 用户确认）

**旧代码**：`z_decay = exp(-z / (2 * h_ink))` = `exp(-z/6μm)` — **完全错误**

**物理分析**：
- EW 力作用在底面疏水层表面（z=0），是油墨与疏水层之间的钉扎力
- z 方向衰减由**德拜屏蔽长度**决定，约 **~50 nm**
- 远小于油墨厚度（~3μm），EW 力是表面力

**修正**：
```python
lambda_d = self.materials_params.get("lambda_debye", 50e-9)  # 德拜屏蔽长度 ~50nm
z_decay = torch.exp(-z_coord / lambda_d)
```

**参数来源**：`lambda_debye = 50e-9` 加入 `PHYSICS` dict 和 `PhysicsConfig` dataclass。

**采样分析**：2000 个物理点中，落在 z < 100nm 范围内的期望数量 = 2000 × (100nm/20μm) ≈ **10 个点**。虽然不多，但足够产生梯度信号。

### 3.3 CSF 表面张力 f_st（✅ 用户确认不改）

**公式**：`f_st = σ * kappa * ∇φ`，`kappa = -div(∇φ/|∇φ|)`

**验证**：>45% 开口率时边缘油膜会聚，和物理一致。公式方向正确，不改。

### 3.4 EW 变分残差（✅ 用户确认关闭）

**旧代码**：`residual = p_ew * phi_b * interface_b`，`interface_b = exp(-100*(phi-0.5)²)`

**问题**：
- `interface_b` 只在 phi≈0.5 有效
- 如果模型没学好界面（phi 到处接近 0 或 1），权重处处接近零，约束失效
- 用户确认：开口区域可能有油膜残留，惩罚 phi→1 不合理

**操作**：`electrowetting_weight = 0`（`config/v4.6-optimized.json`）

### 3.5 器件物理（用户确认）

- **器件类型**：电润湿（非 EPD），癸烷+5%染料油墨
- **油墨密度**：rho_oil = 763 kg/m³（比极性液体 998 略小）
- **油墨粘度**：mu_oil = 9.41e-4 Pa·s
- **接触线以内**（r < r_open）：极性液体润湿底面 → phi→0
- **接触线以外**：油墨堆积 → phi→1
- **>45% 开口率**：表面张力克服毛细力，油膜收向角落
- **关态**：V_eff=0，EW 力消失，只有 f_st 驱动油膜反向铺展

## 4. 训练策略调整

### 4.1 Interface loss 权重退火（`pinn_two_phase.py`）

**旧代码**：`interface_loss * 500 * stage1_factor`，stage1_factor 从 epoch 4000 才开始退火

**修正**：
- S1 (0 ~ stage1_epochs): factor = 1.0 → interface_weight = 500
- S2 (stage1_epochs ~ stage2_epochs): factor 退火 1.0 → 0.1 → interface_weight 500→50
- S3 (stage2_epochs ~ end): factor 退火 0.1 → 0.02 → interface_weight 50→10
